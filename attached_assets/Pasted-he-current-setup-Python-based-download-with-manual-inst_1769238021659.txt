Path 1: Tauri (Frontend) + Python in Background (Recommended – Lighter & Future-Proof)
Tauri builds a tiny native shell (~15-30 MB) that opens the chat window and spawns your Python agent logic as a hidden subprocess. This is a proven pattern (multiple open-source examples exist).
What the User Experiences:

Download → .dmg (nice installer) or .app.zip.
Drag .app to Applications → double-click (or double-click .agentforge file).
Native window opens: avatar at top, chat history, prompt field at bottom.
Python runs invisibly (no console window).
First time / no model: Native dialog (or custom window) with avatar image + "Connect Ollama or API key" fields.

What Devs Need to Do (Key Additions to Current Setup):

Build the AgentForge Runner as Tauri app (if not already started):
Use Tauri v2.
Frontend: Reuse your web chat UI (HTML/JS/CSS) in webview.
Register .agentforge file association (Info.plist for Mac).

Spawn Python as background subprocess:
In Rust code (main.rs): On app launch or file open → use tauri-plugin-shell to spawn python3 agent_runner.py --config /path/to/file.agentforge.
Hide console: Use creation_flags or platform-specific flags to detach/no console.
Communication: Python starts a local FastAPI server (localhost:someport) → Tauri frontend fetches it for chat messages.
