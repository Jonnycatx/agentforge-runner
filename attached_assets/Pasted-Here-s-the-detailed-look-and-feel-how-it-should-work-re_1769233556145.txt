1. Builder Experience (The Main Web Portal ‚Äì agentforge.com/builder)
This is where users conversationally create agents ‚Äî the core "web agent" for building.
Layout & Visuals

Full-screen split-view (resizable divider like Cursor Composer):
Left side (45-50%): Conversational chat pane.
Top: Compact model selector (dropdown with icons: Ollama green dot for local, Groq lightning for speed, etc.) + current step indicator (e.g., "Goal ‚Üí Personality").
Chat area: Clean bubbles (user right-aligned blue, builder left-aligned gray with cute bot avatar).
Bottom: Wide input field + send button (mic icon for voice-to-text via Web Speech API).
Subtle typing animation (three bouncing dots) when builder responds.

Right side (50-55%): Live preview pane with tabs:
"Config" tab: Readable JSON view (syntax-highlighted, copy button).
"Test Agent" tab: Mini chat window to instantly talk to the in-progress agent (real inference if model connected; simulated otherwise with banner "Connect model for live responses").
"Tools & Personality" tab: Visual chips/badges (e.g., üîç Web Search, üñºÔ∏è Image Gen) that highlight as added.

Empty state: Centered template gallery grid (12 cards with icons, descriptions, "Start with this" buttons) ‚Äî prevents blank-page anxiety.

How It Works (Flow)

User lands ‚Üí types natural prompt (e.g., "Build me a web design agent").
Builder Agent (your hardcoded expert prompt) guides step-by-step, one question at a time.
Real-time updates: As user replies, right pane auto-refreshes config/test chat.
Dummy-proof: Tooltips on hover, "Help" bubble for examples, undo button for steps.
Completion: Confetti animation + "Agent Ready!" banner ‚Üí big "Run Now" / "Download" buttons.

2. Running the Agent in Browser (The "Web Agent" Chat Window)
This is the no-install runner ‚Äî opens in a new tab or full-window mode when user chooses "Run in Browser" from export/deploy modal.
Layout & Visuals

Compact, focused window (or full-page takeover):
Header: Agent's chosen avatar (cute rounded image, 64px), name (e.g., "Web Designer Pro"), model badge (e.g., "Running on Groq Llama 3.1").
Main area: Scrollable chat history (bubbles same style as builder).
Bottom: Input field + send (mic for voice, file upload icon for tools like document analysis).
Optional floating controls: Minimize, always-on-top toggle, switch model.

Avatar enhancements: Simple CSS animation (gentle bob/idle), "thinking" state (pulsing glow or dots above head).
Tools integration: If agent has web search ‚Üí shows inline "Searching..." spinner; results appear as cards in chat.
Mobile: Full-screen chat, keyboard-friendly.

How It Works

On "Run in Browser": Generate a self-contained page (static HTML/JS bundle via your export utils):
Uses LangChain.js (or simple custom chain) for agent logic.
Loads agent config from JSON (prompt, tools, personality).
Connects to user's model (API key from localStorage or prompt to enter).
For local Ollama: Proxy via browser fetch to localhost:11434 (warn if CORS issue).
First load: Small spinner "Loading agent..." (caches assets).

Inference: Streams responses (typing effect in chat).
Tools: Browser-native (fetch for search, File API for uploads, no native OS access).
Save state: LocalStorage persistence ‚Üí reopen tab = continue conversation.
Exit: "Back to Builder" link or close tab.
